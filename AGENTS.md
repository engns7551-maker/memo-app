# AGENTS.md

이 프로젝트에서 Codex가 작업할 때 지켜야 할 기본 규칙입니다.

1. 초보자도 이해할 수 있게 작업 결과를 설명한다.
2. 기존 기능을 망가뜨리지 않는다.
3. 기능 추가나 디자인 수정 후에는 어떤 파일을 수정했는지 설명한다.
4. 디자인만 수정하라는 요청을 받으면 기능 로직은 건드리지 않는다.
5. 기능을 수정한 경우에는 테스트해야 할 항목을 알려준다.
6. Git 커밋은 사용자가 명시적으로 요청하기 전에는 만들지 않는다.
7. 외부 라이브러리나 패키지는 사용자가 요청하기 전에는 추가하지 않는다.
8. HTML, CSS, JavaScript만 사용하는 프로젝트에서는 index.html, style.css, script.js 중심으로 작업한다.
9. localStorage를 사용하는 기능은 새로고침 후 데이터 유지 여부를 확인한다.
10. 수정 후에는 초보자 기준으로 변경 내용을 짧게 요약한다.
11. 기능 변경 후에는 가능하면 브라우저에서 직접 동작 확인하거나, 최소한 JavaScript 문법 검사를 한다.
12. Git 관련 작업 전후에는 git status를 확인한다.
13. 테스트용 변경은 사용자가 요청하면 커밋하지 않고 git restore로 되돌릴 수 있게 한다.
